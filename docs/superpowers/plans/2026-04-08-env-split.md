**Tech Stack:** Node.js + TypeScript + dotenv + dotenv-cli；Swift + SwiftUI + #if DEBUG

```typescript

```

```typescript

if (process.env.NODE_ENV === 'development') {
```

文件其余部分（`export const config = { ... }`）保持不变。

## Task 4: iOS — 修改 Config enum 支持 #if DEBUG

**Files:**
- Modify: `ios/HealthSync/HealthSync/Core/DI/AppContainer.swift:68-79`
- Modify: `ios/HealthSync/HealthSync/Modules/Settings/Presentation/ViewModels/SettingsViewModel.swift:100`

- [ ] **Step 1: 修改 `AppContainer.swift` 中的 `Config` enum**

将文件末尾（第 68-79 行）的：
```swift
enum Config {
    private static var cachedServerURL: String?

    static var serverURL: String {
        if let cached = cachedServerURL {
            return cached
        }
        let value = UserDefaults.standard.string(forKey: "serverURL") ?? "https://markmager.cc/healthsync"
        cachedServerURL = value
        return value
    }
}
```

替换为：
```swift
enum Config {
    /// Debug build 默认连接本地 Mac，Release build 默认连接生产服务器
    /// 修改 DEBUG 分支的 IP 为你 Mac 的局域网 IP（用 ifconfig 查看）
    static let defaultServerURL: String = {
        #if DEBUG
        return "http://192.168.124.81:3000"
        #else
        return "https://markmager.cc/healthsync"
        #endif
    }()

    /// UserDefaults 中有覆盖值时使用覆盖值，否则用 defaultServerURL
    static var serverURL: String {
        UserDefaults.standard.string(forKey: "serverURL") ?? defaultServerURL
    }
}
```

> `cachedServerURL` 被移除，原因：它会导致 Settings 修改 URL 后 `Config.serverURL` 仍返回旧缓存值；`UserDefaults` 自身有高效缓存，无需二次缓存。

- [ ] **Step 2: 修改 `SettingsViewModel.swift` 第 100 行的 loadSettings 默认值**

将：
```swift
serverURL = UserDefaults.standard.string(forKey: "serverURL") ?? "http://localhost:3000"
```

替换为：
```swift
serverURL = UserDefaults.standard.string(forKey: "serverURL") ?? Config.defaultServerURL
```

这样 SettingsViewModel 的初始值与 `Config.serverURL` 行为一致，避免两处硬编码不同步。

- [ ] **Step 3: 查看 Mac 局域网 IP，按需更新 Debug 默认值**

在终端运行：
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

取 WiFi 网卡的 IP（通常是 `192.168.x.x` 格式，形如 `192.168.124.81`）。

如果 IP 与代码中的不同，更新 `AppContainer.swift` 中 `#if DEBUG` 分支的值。

> **提示：** Mac IP 会随 DHCP 变化。可在路由器或 Mac 网络设置中为此 Mac 分配静态 IP，以减少频繁修改。

- [ ] **Step 4: 在 Xcode 中用 Debug scheme 构建并验证**

1. 打开 Xcode：`open ios/HealthSync/HealthSync.xcodeproj`
2. 确认 scheme 为 Debug（默认就是）
3. 构建到真机（⌘+R）
4. 打开 App → Settings 界面，确认 Server URL 字段显示 `http://192.168.124.81:3000`（本地 IP）
5. 或在登录页直接尝试登录，观察请求是否打到本地服务端（本地服务端终端会有日志输出）

- [ ] **Step 5: 验证 Release scheme 不受影响**

在 Xcode 中切换到 Release scheme（Product → Scheme → Edit Scheme → Run → Build Configuration: Release），重新构建，确认 `Config.defaultServerURL` 为 `https://markmager.cc/healthsync`。
